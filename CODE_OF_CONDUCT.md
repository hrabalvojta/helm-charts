# Code of Conduct

## Our Standard

This project expects professional, respectful, and technically constructive collaboration.

Examples of behavior that contribute to a healthy environment include:

- Giving actionable, technically grounded feedback without personal attacks
- Assuming good intent while still being precise about defects and risks
- Accepting correction gracefully and updating your position when evidence changes
- Keeping discussions focused on code, operations, documentation, and user impact
- Helping others understand repository conventions instead of gatekeeping them

Examples of unacceptable behavior include:

- Harassment, discrimination, threats, or intimidation
- Personal insults, sarcasm used as a weapon, or repeated hostile behavior
- Public or private abuse based on experience level, identity, or background
- Deliberately disruptive conduct, bad-faith review behavior, or derailing technical discussions
- Publishing private information without explicit permission

## Enforcement Responsibilities

Project maintainers are responsible for clarifying and enforcing this Code of Conduct. They may remove, edit, or reject comments, commits, issues, pull requests, and other contributions that violate these standards.

## Scope

This Code of Conduct applies within all project spaces, including:

- GitHub issues, pull requests, reviews, and discussions
- Release notes, documentation, and commit messages
- Any public or private communication that is reasonably connected to this repository

Maintainers may apply any action they judge appropriate, including:

- Requesting corrective action
- Deleting comments or other content
- Closing issues or pull requests
- Temporarily or permanently restricting participation
